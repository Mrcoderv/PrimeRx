# PrimeRx Enhancement Plan - Bulk Purchases & Supplier Management

## Objective
Enable efficient bulk entry of medicines from purchase invoices (e.g., 25+ items), supplier-wise tracking, due integration, and proper admin management.

## Current Limitations
- Medicines added one-by-one via "Add New Medicine".
- No dedicated Supplier entity or bulk purchase workflow.
- Purchases not systematically linked to Due module or history view.

## Proposed Features

### 1. Supplier Management (Admin)
- **New Model**: `Supplier.cs`
  - Id, Name, PAN, DDA Reg No., Address, Phone, Email, CreditDays, ContactPerson, IsActive, Notes.
- **Pages**:
  - `/Admin/Suppliers/Index.cshtml` - List all suppliers with search/filter.
  - `/Admin/Suppliers/Add.cshtml` - Form to add new supplier.
  - `/Admin/Suppliers/Edit.cshtml` - Edit existing.
- Admin menu link: "Manage Suppliers".

### 2. Bulk Purchase Entry
- **New Page**: `/Purchases/AddBulk.cshtml` or under Inventory > New Purchase.
- Features:
  - Dynamic table for adding multiple rows.
  - Fields per item: Medicine Name, Generic Name, Manufacturer, Batch No., Expiry Date, Qty, Purchase Rate, MRP, Discount %, HS Code (optional).
  - Option to create new medicine automatically if not found.
  - Supplier dropdown (with "Add New" quick link).
  - Invoice No., Date, Total calculation (auto).
  - Payment Type: Cash / Credit → Auto create Due if Credit.
- "Save Purchase" button:
  - Creates/updates Inventory stock.
  - Records Purchase Transaction.
  - Updates Due balance if applicable.

### 3. Purchase History / Transactions
- **New Page**: `/Purchases/Index.cshtml`
  - List all purchases with filters (Supplier, Date Range, Invoice No.).
  - Columns: Date, Invoice No., Supplier, Total Amount, Items Count, Payment Status.
  - Drill-down to view items in that purchase.
- Export to Excel/PDF.

### 4. Integration Points
- **Due Module**: Auto-generate due record for credit purchases.
- **Inventory**: Stock updated in real-time with batch/expiry tracking.
- **Reports**: New "Purchase Reports" section.

### 5. UI/UX Improvements
- Consistent with existing blue theme.
- JavaScript for dynamic row addition and total calculation.
- Paste support from bill (future enhancement).

## Technical Implementation Steps

1. Create `Supplier` model + migration.
2. Add Supplier CRUD pages under Admin.
